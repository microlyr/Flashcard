# JavaScript 核心概念

## 什么是闭包（Closure）？

闭包是指一个函数能够记住并访问它的词法作用域，即使这个函数在它的词法作用域之外被调用。闭包使得函数可以"记住"创建时的环境变量。

## 解释事件循环（Event Loop）

事件循环是 JavaScript 的异步执行机制。它不断地从任务队列中取出任务执行。

- **微任务队列**（Microtask）：Promise.then、MutationObserver
- **宏任务队列**（Macrotask）：setTimeout、setInterval、I/O
  执行顺序：同步代码 → 微任务 → 宏任务（每次一个宏任务后清空所有微任务）

## 什么是原型链（Prototype Chain）？

JavaScript 中每个对象都有一个内部属性 [[Prototype]]，指向它的原型对象。当访问一个对象的属性时，如果对象本身没有，就会沿着原型链向上查找，直到找到或到达 null。

## var、let、const 的区别

| 关键字   | 作用域   | 可重新赋值 | 暂时性死区 | 变量提升          |
| ----- | ----- | ----- | ----- | ------------- |
| var   | 函数作用域 | ✅     | ❌     | ✅ (undefined) |
| let   | 块级作用域 | ✅     | ✅     | ✅ (TDZ)       |
| const | 块级作用域 | ❌     | ✅     | ✅ (TDZ)       |

## 什么是防抖（Debounce）和节流（Throttle）？

**防抖**：在事件被触发 n 秒后再执行回调，如果在这 n 秒内又被触发，则重新计时。适合搜索框输入。
**节流**：规定在一个单位时间内，只能触发一次函数执行。适合滚动事件、resize。

## 解释 this 的四种绑定规则

1. **默认绑定**：独立调用 → this 指向全局（严格模式为 undefined）
2. **隐式绑定**：obj.foo() → this 指向 obj
3. **显式绑定**：call / apply / bind → 明确指定 this
4. **new 绑定**：new Foo() → this 指向新创建的对象
   优先级：new > 显式 > 隐式 > 默认

## 什么是 Promise？有哪些状态？

Promise 是异步编程的一种解决方案。

- **pending**（进行中）→ **fulfilled**（已成功）或 **rejected**（已失败）
- 状态一旦改变就不可逆
- .then() 返回新的 Promise，支持链式调用
- Promise.all / Promise.race / Promise.allSettled / Promise.any

## 什么是深拷贝和浅拷贝？

**浅拷贝**：只复制对象的引用，不复制嵌套对象。方法：Object.assign、展开运算符
**深拷贝**：递归复制所有层级，新对象与原对象完全独立。方法：JSON.parse(JSON.stringify())、structuredClone()、递归实现

## HTTP 状态码分类

- **1xx**：信息响应
- **2xx**：成功（200 OK、201 Created、204 No Content）
- **3xx**：重定向（301 永久、302 临时、304 未修改）
- **4xx**：客户端错误（400 Bad Request、401 未授权、403 禁止、404 未找到）
- **5xx**：服务端错误（500 内部错误、502 网关错误、503 服务不可用）

## 什么是跨域？如何解决？

浏览器的同源策略限制了不同源之间的资源访问。解决方案：

1. CORS（跨域资源共享）：服务端设置 Access-Control-Allow-Origin 头
2. JSONP：利用 script 标签不受同源策略限制（仅 GET）
3. 代理服务器：开发时用 webpack-dev-server / vite proxy
4. postMessage：跨窗口通信
